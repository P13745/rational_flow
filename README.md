# Rational Flow

Rational Flow は、周波数比から少しずつ音を派生させる生成音楽アプリです。

最初の seed または drone から始まり、現在鳴っている音を parent として選び、`3/2` や `5/4` のような有理数比を掛けることで child を作ります。音は画面上を右から左へ流れ、同時に鳴っている音の比率や、100 cents 未満の近接音程を確認できます。

## 公開ページ

GitHub Pages で公開する場合は、このリポジトリの `index.html` をそのまま使えます。

## 開発時の起動

このアプリは静的ファイルだけで動きます。ローカルで確認するときは、リポジトリ直下で簡易サーバーを起動します。

```bash
python3 -m http.server 8000
```

その後、ブラウザで `http://localhost:8000/` を開きます。ファイルを直接開くこともできますが、今後 ES modules へ移行するため、開発時はローカルサーバー経由で確認する方針です。

レファクタリング時の回帰確認項目は [docs/refactoring-checklist.md](docs/refactoring-checklist.md) にまとめています。構成の現状メモは [docs/current-structure.md](docs/current-structure.md) を参照してください。

## 現在の対応環境

デスクトップブラウザに加えて、スマートフォン幅では、設定・メイン画面・リストを切り替える簡易レイアウトを用意しています。モバイルでも主要な再生操作、Diesis List、Help、タイムライン確認ができる構成です。

iPhone Safari では、消音モード中の再生やバックグラウンド再生は保証していません。再生中は対応ブラウザで Screen Wake Lock API を使い、画面の自動ロックを抑制しようとしますが、OSや省電力設定によって解除される場合があります。これらはWeb版では制限が大きいため、将来的にNative app化する場合の検討事項とします。

長時間の再生向けに、タイマーを設定できます。タイマーが有効な場合、終了時刻に近づくにつれて新しい音の生成確率を下げ、指定時間を過ぎると新規生成を止めて残っている音だけを流します。タイマー終了後は画面ロック抑制も解除します。

内部の厳密比率は、巨大な分子分母を直接持つのではなく素因数の指数として扱います。通常は `Ratio` 表示で通分済み整数比を表示しますが、整数が JavaScript の安全な整数範囲を超えそうな場合は、自動的に `Factors` 表示へ切り替わります。`ratio integer limit` を大きくすると候補比率は増えますが、長時間再生時の表示や処理が重くなる場合があります。

## 保留中のアイデア

### Diesis List の検索

Diesis List には、将来的に検索機能を追加したいと考えています。

- 名前や別名による検索。
- 比率そのものによる検索。
- `(3^2*5/7*2)^2` のような数式を含む比率入力の検索。

数式検索では、入力された比率を計算して cents を表示し、既存の Diesis List に該当するものがあればその項目へ移動できるようにします。該当する名前付き diesis がない場合でも、計算結果の比率と cents だけは確認できるようにする方針です。

### Diesis の倍音アルペジオ試聴

Diesis 一覧では、一時的に `Arp` ボタンを試していました。これは comma/diesis の 2 音を鳴らしたまま、そこから推定される基音の倍音列をアルペジオとして重ねる機能です。

このアイデア自体は面白いのですが、現時点では保留しています。名前付き comma には `531441/524288` や `2109375/2097152` のような非常に大きな比率もあり、それらをそのまま倍音番号として oscillator に展開すると、音楽的にも実装的にも扱いが難しくなります。

- 予約する oscillator が多すぎるとブラウザへの負荷が高い。
- 自動的に間引くと、どの倍音を残すかによって意味が変わってしまう。
- 推定される基音が低すぎて、ほとんど聞こえない場合がある。
- 密なアルペジオが、聞きたいはずの小さな音程差を隠してしまう。

将来もう一度入れるなら、次のような方向が良さそうです。

- 文字通りの全倍音ではなく、少数の知覚的なガイドトーンとして鳴らす。
- 倍音ごとに oscillator を立てるのではなく、offline buffer や加算合成でまとめて扱う。
- sparse / odd harmonics / prime harmonics / octave anchors など、複数の試聴モードとして分ける。
- これは厳密な再現ではなく解釈的なプレビューであることを、UI上でも分かるようにする。
